## 1. What Must Exist for Login to Work

The login flow touches **exactly one table**: `users`.

### Login code path (`app.py` `auth_login`, line 824)

| Step | Query | Table | What's Needed |
|------|-------|-------|---------------|
| 1 | `User.query.filter_by(email=email)` | `users` | At least 1 user record |
| 2 | `check_password(password, user.password_hash)` | `users` | `password_hash` column with bcrypt hash |
| 3 | `user.is_verified` check | `users` | `is_verified = true` |
| 4 | Session set: `session['user_id'] = user.id` | (session) | Filesystem session, no DB dependency |
| 5 | `user.role == 'admin'` redirect | `users` | `role` column |

### Minimum record for login

```sql
-- Equivalent SQL for a regular user who can log in
INSERT INTO users (email, first_name, last_name, password_hash, role, is_verified, created_at)
VALUES (
  'user@example.com',
  'Test',
  'User',
  '$2b$12$...',  -- bcrypt hash
  'user',          -- role
  true,            -- is_verified
  NOW()
);
```

**No other tables are queried during login.** Not even `colleges`, `college_cutoffs`, or any admin tables.

### Admin login requirement

Admin login (`admin/routes.py` line 94) adds one filter:
```python
User.query.filter_by(email=email, role='admin').first()
```
So the admin record must have `role='admin'` in addition to `is_verified=true`.

## 2. What Admin Records Must Be Seeded

### Source: `app.py` lines 989-1013

The app seeds an admin **only from environment variables** at startup:

```python
admin_email = os.environ.get('ADMIN_EMAIL', '').strip().lower()
admin_password = os.environ.get('ADMIN_PASSWORD', '').strip()
if admin_email and admin_password and len(admin_password) >= 8:
    existing_admin = User.query.filter_by(email=admin_email).first()
    if not existing_admin:
        admin_user = User(
            email=admin_email,
            first_name='Admin',
            last_name='User',
            password_hash=hash_password(admin_password),
            role='admin',
            is_verified=True,
            created_at=datetime.utcnow(),
        )
        db.session.add(admin_user)
        db.session.commit()
```

### Required environment variables

| Variable | Example Value | Notes |
|----------|---------------|-------|
| `ADMIN_EMAIL` | `admin@example.com` | Must be set and non-empty |
| `ADMIN_PASSWORD` | `MyS3cur3P@ss!` | Must be ≥ 8 characters |

### SQL equivalent of the seeded admin record

```sql
INSERT INTO users (email, first_name, last_name, password_hash, role, is_verified, created_at)
VALUES (
  'admin@example.com',
  'Admin',
  'User',
  '<bcrypt_hash_of_admin_password>',  -- generated by hash_password()
  'admin',
  true,
  NOW()
);
```

### What happens if env vars are NOT set?

- No admin user is created.
- `/admin/*` routes (protected by `@admin_required` → `user.is_admin()` which checks `role == 'admin' AND is_verified`) will all redirect to login.
- **You cannot access any admin functionality** until an admin user exists.

### What if a user with that email already exists?

```python
elif not existing_admin.is_admin():
    existing_admin.role = 'admin'
    existing_admin.is_verified = True
    if not existing_admin.password_hash:
        existing_admin.password_hash = hash_password(admin_password)
```
The existing user is promoted to admin. No duplicate is created.

### 3b. `init_sample_cutoff_data()` — Multi-year, multi-category cutoff records

Called at startup from `app.py` line 987:
```python
init_sample_cutoff_data()
```

This inserts into `college_cutoffs` table **only if it's empty** (`CollegeCutoff.query.count() == 0`).

**What it generates for each of the 30 Maharashtra-located colleges (Mumbai/Pune/Nagpur):**
- 3 years: 2022, 2023, 2024
- 5 categories: Open, OBC, SC, ST, EWS
- 2 genders: Male, Female
- Base percentile driven by college rank (IIT → 99%, NIT → 92%, etc.)
- Category adjustments: Open ±0, OBC -8, SC -20, ST -25, EWS -5
- All marked `approval_status = 'approved'`

**Approximate row count:** 30 colleges × 3 years × 5 categories × 2 genders = **900 rows**

**Without this seed data:**
- MH-CET recommender (`mhcet_recommender.py`) returns empty results
- College profile page (`/college/<id>`) shows no cutoff history
- Trends page has no data to analyze

## 5. Verify No Hidden Dependencies on Old Data

### Checked and verified — NONE found

| Risk Area | Investigation | Verdict |
|-----------|--------------|---------|
| **Migration scripts** | `migrate_unified_cutoffs.py` exists but is a standalone tool, not called at startup | **SAFE** |
| **Old `cap_cutoffs` table** | Only queried in admin dashboard for count display; FK `college_id` → `colleges.id` is NOT NULL but if no data is inserted, no violation | **SAFE** — can be empty |
| **Legacy `College` model data** | `CAPCutoff` FK requires `colleges.id` to exist, but `init_sample_data()` creates 12 colleges, satisfying this | **SAFE** |
| **SQLite (instance/) files** | `instance/collegekhoj.db` is local dev only. Neon uses its own PostgreSQL, no file dependency | **SAFE** |
| **Session store** | `SESSION_TYPE = "filesystem"` — no database table needed for sessions | **SAFE** |
| **Email service** | `email_service.py` calls Resend API. If API key is missing, email fails but login/signup still works (code is generated locally) | **SAFE** |
| **Auth before `db.create_all()`** | `auth_login` queries `User` table which requires `users` to exist, but `db.create_all()` runs before any requests can be served (it's in the `with app.app_context():` block at module level) | **SAFE** |
| **Recommender engine** | Requires `College` table (seeded by `init_sample_data()`) and `CollegeCutoff` table (seeded by `init_sample_cutoff_data()`). Both check for empty table before inserting | **SAFE** |
